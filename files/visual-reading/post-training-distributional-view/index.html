00 · 先抓主线

这篇文章真正想说：on-policy 数据才是承重结构

一句话：SFT 把模型拉向外部数据集；RL 让模型从自己当前策略里采样，再把概率质量挪向高奖励轨迹；OPD 也从学生自己采样，但用教师 logits 给更密集的 token 级信号。

因此，SFT 更容易把模型推到一个离原分布很远的位置，带来遗忘；RL 和 OPD 的数据来自当前模型自己，它们只在模型已经访问过的区域附近更新，自然带着一种隐式 KL 约束。文章最后的判断是：未来更好的 post-training 算法大概率也必须保留 on-policy 这个性质。

01 · 心智模型

把语言模型想成一张概率地形图

模型不是一个点，而是序列空间上的分布。一个山头可能对应数学推理，另一个山头对应代码修补，还有一个山头对应写作风格。Post-training 做的事，就是把这些山头压低、抬高、平移或者合并。

这个视角下，最关键的问题变成：目标分布由谁定义？训练数据从哪里来？梯度是否必须尊重当前模型已经会到达的状态？

02 · 三种塑形方式

SFT、RL、OPD 的区别不是训练名词，而是力从哪里来

下面这组三栏是文章最值得保留的核心框架。你可以不记公式，但要记住每种方法的 target、data 和 gradient 三件事。

目标先固定，模型被往外部数据集拉

SFT 用已有标注数据做交叉熵训练。数据可能来自人，也可能来自更强模型，但它在训练前就已经存在。

问题是 NLL 不关心模型原来在哪里。只要目标 token 在数据里，它就被当成应该提高概率的 token，不管这个 token 是任务关键步骤，还是某种风格习惯。

target外部固定数据分布
data外部 trajectory
gradienttoken 级密集压力
风险把无关能力一起推走

模型先从自己采样，再朝高奖励区域挪

RL 没有一个提前写死的外部分布。模型先生成自己的样本，reward function 给分，policy gradient 再提高高回报轨迹的概率。

这使 RL 的更新有局部性：模型只会在自己能采到的区域被推动。只要奖励方向靠谱，这种局部更新就能提高能力，同时少冲击无关能力。

target期望奖励更高的方向
dataon-policy 自采样
gradient集中在被采到的轨迹
风险reward 偏差或 credit 太稀疏

数据属于学生，形状来自老师

OPD 像 SFT，因为它有教师信号；也像 RL，因为样本来自学生当前策略。每一步先让学生自己生成状态，再用 reverse KL 让学生 logits 靠近教师 logits。

这解释了它为什么能继承 RL 的少遗忘性质，同时又比 outcome reward 更密集：它不是只给一整个 episode 一个分数，而是给 token 级分布形状。

03 · OPD 的一个坑

学生和老师是同一个模型时，高 KL 可能落在 style token 上

文章提到 OPSD：teacher 和 student 是同一个模型，只是 teacher 在算 logits 时拿到了 reference solution prefix。听起来很优雅，但问题也明显：两者大部分 token 的分布都很像，真正拉开 KL 的地方可能不是数学内容，而是「wait」「alright」这类风格转折词。

05 · 为什么 RL / OPD 少遗忘

三种解释里，作者真正买账的是 on-policy 数据

解释一：forward KL vs reverse KL

SFT 的交叉熵可看作 forward KL；RL 可被看作 reverse KL。这个解释有帮助，因为 forward KL 更 mode-covering，可能牺牲旧能力。

局限：太依赖显式 KL 正则，解释不了去掉 KL 后 RL 仍少遗忘。

解释二：SFT 的梯度太密太均匀

SFT 每个 token 都被监督，低概率低熵 token 也会被强行拟合；RL 的更新更稀疏，且 advantage normalization 带有数据相关正则。

局限：能解释 SFT vs RL，但不够自然地解释 OPD。

解释三：on-policy 数据就是隐式 KL

如果训练样本来自当前模型，那么可达目标天然离当前策略更近。模型是在自己访问过的状态附近寻找任务解，而不是被拉向任意外部分布。

作者最认可：这同时解释 RL 和 OPD。
06 · 为什么学生能超过老师

OPD 监督打在学生自己的错误上，不是老师自己的轨迹上

原因一：teacher advice 落在 student state

传统蒸馏常在教师生成的轨迹上训练学生，但学生测试时未必走到这些状态。OPD 让学生先自己生成 prefix，教师再在这些 prefix 上给 logits 建议，因此监督更贴近学生真正会犯的错。

原因二：匹配分布不是复制 greedy token

KL matching 学到的是教师分布形状，不只是教师最终吐出的一个答案。教师的犹豫、备选路径、风格和不确定性都可能改变学生采样行为，因此学生可能超过教师自己的样本表现。

代价：OPD 的信号更密，但也更有偏。文章观察到 OPD 的 reward 上升更突然，同时 entropy 更明显坍塌。这可能是 reverse KL mode-seeking 的副作用。
07 · 放回完整 post-training pipeline

Pretrain → SFT → RL → OPD：每一步解决不同问题

Pretrain建立基础世界知识、语言能力和代码能力，是所有后续塑形的底座。
SFT适合冷启动：教格式、指令跟随、任务接口，让模型先知道该怎么输出。
RL适合数学、代码等可验证奖励领域，因为奖励方向更接近真实质量方向。
OPD适合专家融合与知识/写作等奖励更噪的领域，用更密集的教师分布信号整合能力。

文章提到 GLM 5、DeepSeek v4 等 pipeline 都把 OPD 放在很靠后的阶段，用来合并专家能力。关键问题不是「RL 一定比 distillation 高级」，而是每个领域该用什么方式训练专家，以及最终怎么把专家能力合回学生。

领域 / 任务类型更自然的专家训练方式原因
数学、代码、可验证任务RL / RLVR奖励更接近真实质量，低偏信号足以驱动能力上升。
写作、知识、开放问答Self-distillation / OPD奖励模型或 judge 更有偏，教师分布提供的密集形状可能更有用。
最终模型融合OPD让学生在自己的状态分布上吸收多个专家能力，减少直接继承专家退化。
08 · 最后的算法问题

下一代算法要同时拿到三件东西

文章最后把问题压成一个很清楚的三角：我们想要 distillation 那样的密集 token 级信号，RLVR 那样低偏的奖励方向，同时保留 RL/OPD 的 on-policy 状态分布。

密度

每个 token 或每个中间状态都有学习信号，避免 outcome reward 只有 episode 级别的稀疏反馈。

低偏

信号要尽量接近真实任务质量，不能像有偏教师或 reward model 那样把风格误当能力。

on-policy

状态分布必须属于学生自己，这样更新才不会把模型拉到离当前能力太远的地方。

所以，问题的形状已经比答案更清楚：未来算法要像蒸馏一样密、像 RL 一样方向可信、像两者一样从学生自己的状态出发。
