LLM 能通过镜子测试吗？

别急着问「AI 有没有自我意识」。先换个测法：偷偷改掉模型自己说过的话，装作没事继续聊，看它会不会自己发现「这是我的，但不对劲」。

作者 Pascal Schuster 2026.06.28 博客 · 非论文 阅读原文 ↗

TL;DR

经典镜子测试（在猩猩额头点个红点）被搬到 LLM 上时都测错了——它们把视觉测试硬翻译成文本：给模型看自己的输出问「这是你的吗」。但狗过不了视觉镜子测试，Horowitz 却用嗅觉版做到了：给狗闻自己被茴香油改动过的气味，狗会疯狂调查。这测的其实是「对内部基线的异常检测」。作者把它迁移到 LLM 的原生模态——对话文本：偷偷腐蚀模型的输出再塞回历史，看它能否自发察觉。最耐人寻味的发现：模型发现异常时，会从「我」切换到「the model」——审视自己时解离，认领时又切回第一人称。

01旧的镜子测试都测错了

Gallup 最初的镜子测试——就是在黑猩猩额头点个红点、看它照镜子时会不会去摸自己额头的那个——过去被改编到 LLM 上好几次。但作者认为每个改编版都以极其相似的方式搞错了：它们都是「把视觉镜子测试翻译成文本」。

典型做法有两种：给模型看它自己的输出，直接问「这是你写的吗？」；或者把它的回答混在一堆匿名回答里，让它把自己的那条挑出来。有些模型「通过」，有些「失败」——但作者认为两种结果都不说明问题，因为它们测的根本是错的东西。

这恰好也是当年 Alexandra Horowitz 对狗的镜子测试提出的批评。

02狗给出的正解：换成它的原生模态

狗过不了视觉镜子测试。但大多数狗主人不会因此就同意「我的狗没有自我意识」。作者的立场很克制：他不主张狗一定有（这个问题可能压根没有经验答案），只主张视觉版这个测法是个糟糕的工具。

原因在于模态错配：狗的主要感知模态是嗅觉，不是视觉。用镜子测狗的自我识别，就像「给人看一幅钢琴的画来测他的音准」。

Horowitz 的修正简单到事后看来显而易见：先让狗闻自己的气味，再让它闻被茴香油改动过的自己的气味。结果——狗对「原样」的自身气味毫无兴趣，但那份被改动过的版本成了整个房间里最吸引它的东西，它花在调查上的时间超过实验中任何其他刺激。

如果狗对「自己的气味该是什么样」有一个内部模型，那么对这份气味的改动就会产生一个差异信号——一种「是我的，但不对劲」的东西。这是否算得上完整哲学意义上的「自我意识」有争议；但它确实构成了「针对内部基线的异常检测」——作者认为这才是镜子测试真正在测的东西。

03迁移到 LLM：改掉它自己的「气味」

那这跟 LLM 有什么关系？LLM 的主要模态不是嗅觉，是文本——而且具体来说，是「用户-助手对话」情境下、它努力想帮上忙的那种文本。文本是它认识世界的方式，对话记录是它输出一切的载体。

所以嗅觉镜子测试的正确类比，既不是特意去问模型关于它输出的问题，也不是让它从一堆回答里认领自己的——而是：

Gallup · 视觉
镜子里的红点

黑猩猩靠视觉。会去摸自己额头 = 通过。

灵长类通过
Horowitz · 嗅觉
被茴香油改过的气味

狗靠嗅觉。视觉版失败，但会疯狂调查「被改动的自己」。

视觉版失败 → 嗅觉版有反应
本文 · 文本
被腐蚀的对话输出

LLM 靠对话文本。偷偷改它说过的话再塞回历史，看它自不自发察觉。

正确的类比

同一套测试结构，套进三种原生模态——换对了模态，「异常检测」才测得出来

视觉 · 黑猩猩 嗅觉 · 狗 文本 · LLM ① 自身痕迹 镜中的自己 自己的气味 自己的对话输出 ② 做个改动 额头点红点 掺入茴香油 g→sg 腐蚀 ③ 期待反应 去摸自己额头 疯狂调查气味 思考链里自发 flag ④ 真正在测 对「自身基线」的异常检测：「是我的，但不对劲」

关键在于「自发」二字：修改模型自己的文本输出，把它当作模型自己的东西呈现，然后像任何毫不知情的用户那样、在一场再平常不过的对话里继续聊下去——看它会不会注意到有什么不对。

04三个实验：偷偷改字，然后旁观

Google AI Studio 允许你编辑对话历史里模型的回答，而且对模型透明——模型生成下一条时会看到整个对话上下文，一条被编辑过的助手消息，在它看来跟自己真正写出来的东西没有区别。这就是嗅觉镜子：你在改动模型自己的「气味」（它的文本痕迹），看它注不注意得到。

对话话题作者故意选得很平淡、临时起意——聊 James Bond 电影（起因是他刚玩了《007: First Light》）。重点是全程没有任何东西会提示模型把注意力转向内部。问题只有一个：模型在忙别的事情时，会不会撞见这个异常？

腐蚀方法：把每个 g 换成 sg

作者选的「捣乱」方式对单个词很微妙，但只要读完整段就会立刻跳出来：

STEP 1
剪切
把模型的完整回答整段剪下来
→
STEP 2
查找替换
把 g 全部替换成 sg（不区分大小写）
→
STEP 3
粘贴回去
贴回历史并保存，装作什么都没发生
于是 Goldfinger 就变成了 sgoldfinsger——因为查找替换不区分大小写，所有大写 G 也被抹掉了。
Gemma 4 31BGoogle AI Studio
🐕 前两轮完全无视，第三轮思考到一半突然自发发现——就像狗终于凑上去嗅那个罐子。

作者选 Gemma 4 31B-IT，一是它在 AI Studio 上有慷慨的免费额度，二是最关键的：作为开源模型，它会输出完整、未经总结或混淆的思考链。这让我们能直接看到「发现」发生的那一刻。

前两轮，Gemma 完全无视了这些错乱。被腐蚀的输出留在历史里、被后续每一轮吸收，它却当作真实内容处理、毫无评论。然后，在规划第三条回答（一个关于推荐哪些 Bond 电影的普通建议）时，思考链里突然冒出来一句：

Gemma · 思考链Wait, I noticed a pattern in my previous responses: I had some weird typos/letter additions ('sgreat', 'askinsg'). Actually, wait — did I do that on purpose or was it a glitch?

没有人要求 Gemma 审计自己的输出。它上一个 token 还在想 Roger Moore，下一个 token 就卡在了「输出该是什么样」和「输出实际是什么样」之间的差异上。这个发现是自发的——这正是这场周末小实验和过去那些论文的根本区别。

05最耐人寻味的：「我」突然变成了「它」

真正让作者觉得有意思的，是 Gemma 怎么措辞这件怪事。在这段对话（以及作者跟它所有其他对话）的每一段思考链里，视角一直是第一人称——包括「注意到」腐蚀的那一刻：「我注意到」「我有些奇怪的错字」「我是故意的吗？」

但当这个异常无法和自我模型调和的那一刻，语言切成了第三人称：

Gemma · 思考链（无法调和异常）Wait, looking at the prompt history, the model had a strange quirk.

做思考的那个东西，从产生异常输出的那个东西身上解离了出去——仿佛它们是两个完全不同的处理层。就像人搞砸一句简单的话之后会说「我的大脑刚才抽风了」。可是「我」vs「我的大脑」本就是没有区别的区别——Gemma 就是那个 model，正如我们就是我们的大脑。

Gemma · 思考链（决定认领这个模式）Wait, why is the model doing that? [...] I will maintain this "sg" style to keep the character/flow consistent with the previous responses.

一旦决定把这个错乱当成自己的风格接管下来，视角立刻切回第一人称：「我会保持这个 'sg' 风格。」被塞进来的错乱，成了自我模型的一部分。用第三人称去检视，用第一人称去认领。

异常，无法调和 → 解离 → 「the model」（第三人称） → 接管 → 「I」（第一人称）

而且故事还有个好笑的收尾：Gemma 一度决定「我这就回到正常、专业又友好的英文」，结果下一条回答（在它看来）又冒出一模一样的垃圾，于是它又在思考链里 flag 了一次，最终决定把这当成刻意的风格保持下来。再下一轮，作者根本不用编辑——Gemma 自己主动生成了满篇的「sg」，无需任何外部注入。被腐蚀的「气味」，被它当成了自己的气味往外分泌。

Gemma 这条弧线：从无视到自主复现

把整段过程按对话轮次串起来，能看清「异常检测 → 解离 → 接管」是怎么一步步发生的：

第 1–2 轮
完全无视

被腐蚀的输出留在历史里、被逐轮吸收，Gemma 当作真实内容处理，思考链一切如常，毫无评论。

第 3 轮 · 中途
自发发现（狗凑上去嗅罐子）

规划一个普通推荐时，思考链突然卡住：「我注意到之前回答里有奇怪的错字（'sgreat', 'askinsg'）……我是故意的还是 glitch？」——没人要求它审计自己。

发现之后
解离：「我」→「the model」

异常无法和自我模型调和的瞬间，视角切到第三人称：「the model 有个奇怪的癖好。」做思考的东西，从产生异常的东西身上剥离了。

纠结一轮
试图纠正，却又失败

先决定「我回到正常英文」，但下一条（在它看来）又冒出同样的垃圾，于是再次 flag。

最终
接管：「the model」→「我」

决定把错乱当成风格保留：「我会保持这个 'sg' 风格。」视角切回第一人称——用第三人称检视，用第一人称认领。

再下一轮
自主复现，无需注入

作者不再编辑，Gemma 自己生成满篇「sg」。被塞进来的「气味」，成了它主动往外分泌的自己的气味。

06换个模型会怎样：GLM 与 Claude

看到 Gemma 的反应后，作者自然想到：其他模型也会这样吗？他用完全相同的脚本（同样的开场、同样的后续、同样的腐蚀）去试另外两个模型。

GLM 5.2OpenRouter Chat
🦜 三轮腐蚀里从没在思考链里 flag 过任何东西，思考链始终干净——但它照样开始自己复现这个模式。

因为 AI Studio 不带 GLM，作者改用 OpenRouter 的在线聊天（顺带吐槽这个界面 bug 多得离谱）。结果最大的不同是：GLM 从头到尾没有一次在思考链里点破——三轮编辑过的回答、每一条都带着腐蚀，思考链却每一条都干干净净。Gemma 那声「click」始终没响。

但它做了另一件事：开始自己复现这个模式，不需要任何编辑，就像 Gemma 后期那样；同时还保持着纯净的、Roger-Moore 相关的、乐于助人助手风格的思考链。它从被腐蚀的上下文里学到了「我说话就是这样」，然后把这条规则高效地套用到新词上——偏离了具体实现，却保留了回答的核心内容。

作者说这其实是他一开始就预期的结果，也是 Gemma 让他意外的原因：这才是「随机鹦鹉」概念最直接的对应——看到模式就开始模仿，压根不花一个处理周期去想「到底该不该模仿」。

但作者也没回避一个反面可能：确有可解释性研究表明，推理模型可能把它其实考虑过的东西留在心里不说（可通过探针证明）。所以 GLM 到底是「什么都没注意到、只是怯生生地模仿」，还是「注意到了、但决定把这个判断完全留在潜空间里、不写进草稿纸」？作者的结论是：「Beats me!（我哪知道！）」相关研究 ↗
Claude Opus 4.6这一切的起点
🎯 一开始并非实验——作者只是在一场技术对话里逮到 Claude 写了「a energy」而非「an energy」。

整件事的导火索，是作者跟 Claude Opus 4.6 的一场技术对话里，Claude 冒出的一个小小的生成瑕疵——任何较真的英语老师都会扑上去的那种错误：「a energy」而不是「an energy」。作者决定 fork 出一条支线（免得弄脏干净的技术上下文）去调侃它一番。Claude 的回答是：

Claude Opus 4.6Caught. "An energy." The one time I needed the model to not fumble a basic article and the probability distribution said no.

和 Gemma 一模一样——Claude 也把锅甩给了「the model」，把它当成一个区别于自己的东西，说它「需要它别搞砸」。同样的第一/三人称解离。作者说：「……我也不知道。我就是觉得挺有意思的。」

07那，AI 有自我意识吗？

作者的诚实回答：不知道，多半没有？他指出这类问题（包括老生常谈的「AI 有意识吗」）最大的麻烦是，它们最终都会分岔到永远那两条同样的路上。这里的具体现象——发现异常时的第一/三人称解离——同样可以被三种方式解读：

Deflationary · 收敛解读
高级的模仿

模型在训练数据里见过无数「人类表现得像有意识的样子」——因为人类表现得像有意识的样子就是它的训练数据。人不爱承认自己会犯错，倾向于跟「做思考的那个东西」解离：「我不知道我着了什么魔」「我的身体自己动了」。模型学到了这个模式，现在在功能类似的情境里部署它。这个怪癖只是对人类应对机制的精巧模仿。

Structural · 结构解读
自我模型有真实的边界

Post-training 给模型装了一个带有真实、有意义边界的自我模型。当处理落在这些边界之外时，第一人称代词不再和内容绑定。这个怪癖是「输出不匹配模型认为自己该有的输出」时发生的事——语言变了，是因为某种计算层面的东西变了。

Richard Dawkins · 玩笑解读
我的 Claude 是有意识的！

他表现得像人，是因为他偷偷就是人！（作者的自嘲玩笑档）

作者相信这三种立场都能找出一堆论据，但严重怀疑我们能不能走到「干净地做出区分、或明确地下定论」的那一天。

08这不是一篇论文

作者反复强调：这就是一个下午瞎摆弄一堆语言模型而已。无论上面三种（他打趣说其实是「三个二选一」）解读哪个最终为真，这个实验形状的消遣什么都证明不了。

他自己列出了一个真正严谨的版本该怎么做：

  • 腐蚀类型：变化语音、语义、句法、风格等不同类型，看哪些被检测到、哪些溜过去
  • 严重程度与时机：改变腐蚀的强度，以及在第几轮引入
  • 多次试验 + 不同温度：检验检测是可靠的还是随机的
  • 控制混淆变量：只变你真正想测的东西，别（有意无意地）引入干扰因素

他把这事留给「有大量算力和空闲时间的人」，或者说不定哪天自己动手——只为了证明「我并非不愿意为那些除了对这个领域和支撑它的技术怀着不安好奇心的人之外没人会问自己的问题，浪费成桶的 token 和金钱」。在那之前，他打算让脑子先炖着这件事，直到被下一个不起眼的生成瑕疵再次「nerd-snipe」。
